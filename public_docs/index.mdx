# Orinadus Foundry

Orinadus builds the **neutral memory layer** — infrastructure that keeps every tool, agent, and
system you use in sync with a single source of truth.

**Urchin** is the substrate. It runs locally on your machine, captures everything that matters,
and makes it queryable across every tool in your stack: your IDE, your terminal, your AI agents.

---

## What Urchin does

- Captures events from shell history, git commits, AI chat sessions, agent reflections, and
  any tool that writes to the intake API
- Stores everything in a local append-only JSONL journal — your data, on your machine
- Exposes a 10-tool MCP server so any MCP-compatible IDE (Cursor, Zed, VS Code with Copilot)
  can query your full context history
- Provides a CLI (`urchin`) for manual ingestion, querying, and vault projection

---

## Architecture in one paragraph

Every write path funnels through a single HTTP endpoint (`POST /ingest` on `127.0.0.1:18799`)
or the MCP server's `urchin_ingest` tool. Both paths write to the same append-only JSONL journal.
Collectors run on a tick loop and pull from shell history, git, and third-party AI tools. A Tauri
desktop app provides a unified surface for AI web chat sessions with passive capture baked in.

## Quick install

```bash
# Clone and build
git clone https://github.com/orinadus-systems/urchin-rust
cd urchin-rust
cargo build --release

# Start the daemon
./target/release/urchin serve

# Verify
curl http://127.0.0.1:18799/health
# → {"status":"ok","events":0,"ephemeral":false}
```

---

## MCP setup (Cursor)

Add to `.cursor/mcp.json` in any repository:

```json
{
  "mcpServers": {
    "urchin": {
      "command": "urchin",
      "args": ["mcp"]
    }
  }
}
```

Cursor will pick up the 10-tool server automatically on next restart.
